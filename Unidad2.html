DDL, DML, DCL, TCL.

Se trata de unos componentes de SQL que se componen a su vez de varias sentencias a la hora de crear nuestra base.

¿Qué es DDL?

DDL significa Data Definition Language o Lenguaje de Definición de Datos, en español. Permite crear y modificar la estructura de una base de datos.

Sentencias DDL

  • CREATE: Utilizado para crear nuevas tablas, campos e índices.
  • ALTER: Utilizado para modificar las tablas agregando campos o cambiando la definición de los campos.
  • DROP: Empleado para eliminar tablas e índices.
  • TRUNCATE: Empleado para eliminar todos los registros de una tabla.
  • COMMENT: Utilizado para agregar comentarios al diccionario de datos.
  • RENAME: Tal como su nombre lo indica es utilizado para renombrar objetos.

    • ¿Qué es DML?

    DML significa Data Manipulation Language o Lenguaje de Manipulación de Datos en español. Permite recuperar, almacenar, modificar, eliminar, insertar y actualizar datos de una base de datos.

    Sentencias DML

  • SELECT: Utilizado para consultar registros de la base de datos que satisfagan un criterio determinado.
  • INSERT: Utilizado para cargar de datos en la base de datos en una única operación.
  • UPDATE: Utilizado para modificar los valores de los campos y registros especificados.
  • DELETE: Utilizado para eliminar registros de una tabla de una base de datos.

    • ¿Qué es DCL?

    DCL significa Data Control Language o Lenguaje de Control de Datos en español.Permite crear roles, permisos e integridad referencial, así como el control al acceso a la base de datos.

    Sentencias DCL

  • GRANT: Usado para otorgar privilegios de acceso de usuario a la base de datos.
  • REVOKE: Utilizado para retirar privilegios de acceso otorgados con el comando GRANT.

    • ¿Qué es TCL?

    TCL significa Transactional Control Language o Lenguaje de Control Transaccional en español. Permite administrar diferentes transacciones que ocurren dentro de una base de datos.

    Sentencias TCL

  • COMMIT: Empleado para guardar el trabajo hecho.
  • ROLLBACK: Utilizado para deshacer la modificación que hice desde el ultimo COMMIT.

    • SQL CREATE

    Podemos crear con esta sentencia tanto tablas como la propia base de datos dicha.

    CREATE DATABASE Nombre_Base;

    CREATE TABLE tabla(

    Columna1 tipoDato,

    Columna2 tipoDato,

    Columna3 tipoDato...);

    SQL DROP

    Con la sentencia DROP podemos eliminar elementos de una base de datos, como también la base misma.

    DROP DATABASE Nombre_Base;

    DROP TABLE tabla;

    También existe la sentencia TRUNCATE que nos elimina el contenido de una tabla sin eliminarla

    TRUNCATE TABLE tabla;

    SQL ALTER

    ALTER TABLE se utiliza para eliminar o modificar las columnas de una tabla.

    También se puede emplear para añadir o eliminar constantes a dichas columnas.

    ALTER TABLE tabla

    ADD columna1 tipoDato;

    Con esta sentencia añadimos una columna a la tabla.

    ALTER TABLE tabla

    DROP COLUMN columna1;

    Con esta sentencia eliminamos una columna de una tabla.

    ALTER TABLE tabla

    MODIFY COLUMN columna1 tipoDato;

    Empleamos esta sentencia para cambiar el tipo de dato de la columna.
